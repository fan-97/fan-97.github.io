java集合框架

2019-07-29 java基础学习总结

java.util包下的集合框架

四大集合接口

  • Collection

    Collection 层次结构 中的根接口。Collection 表示一组对象，这些对象也称为 collection 的元素。

  • List
  • Set
  • Map

List

有序可重复Collection集合 可以拥有多个null元素

常用的方法：

  boolean add(E e)  添加元素到列表的尾部
  void add(int index, E element)  向指定位置添加元素
  E get(int index)  返回列表指定位置的元素
  E remove(int index)  移除指定位置的元素

常用子类：

  • ArrayList:
    遍历集合的3中方法
    List<Integer> L1 = new ArrayList<>();
L1.add(1);
L1.add(2);
L1.add(3);
L1.add(4);
//遍历集合：
//1.for循环
System.out.println("----for loop------");
for(int i=0;i<L1.size();i++){
  System.out.print(L1.get(i));
}
System.out.println("-----foreach------");
//2.增强型for jdk8++
for(Integer i : L1){
 System.out.print(i);
}
System.out.println("------Iterator-----");
//3.Iterator
Iterator<Integer> it = L1.iterator();
while(it.hasNext()){
  System.out.print(it.next());
}
public ArrayList() {
  this.elementData = DEFAULTCAPACITY_EMPTY_ELEMENTDATA;
}
transient Object[] elementData; // non-private to simplify nested class access
private static final Object[] DEFAULTCAPACITY_EMPTY_ELEMENTDATA = {};

由构造方法中的两个参数可以看出，ArrayList底层是由数组实现的，默认是创建一个长度为0的Objcet数组
优点：查找数据方便，可以通过索引进行访问
缺点：操作数据麻烦，当要删除某个元素时，要移动它之后的所有元素

  • LinkedList


里面的变量
transient Node<E> first;
transient Node<E> last;
transient:关键字 当序列化的时候忽略这个关键字修饰的变量 不能放在方法体类
只能用来修饰成员变量或者类变量

private static class Node<E> {
E item;
Node<E> next;
Node<E> prev;

Node(Node<E> prev, E element, Node<E> next) {
        this.item = element;
        this.next = next;
        this.prev = prev;
    }
}

LinkedList是对链表进行操作，底层是链表实现，还实现了Deque 可以进行栈对应的操作

push(E) 入栈，采用的是头插法
poll()  将栈顶元素出栈但不移除元素
↑ 方法是ArrayList没有的方法

优点：插入，删除元素快，只需要删除对应的结点，不需要移动元素
缺点：查找元素慢，因为需要从头结点遍历 可以采用双向链表优化

Set

无序不可能重复的Collection集合 最多只能拥有一个null 元素
底层是Map存放key,value键值对，因为Key不能重复，所以set不能存放重复的元素

常用方法(Set接口中的方法):

contains(Object o) : 查询是否集合中是否包含该元素
isEmpty() : 为空返回true
iterator() : Set集合的迭代器
remove(Object o) : 移除某个元素
size() : 集合的大小

常用子类

  • HashSet

    由哈希表（实际上是一个 HashMap 实例）支持。它不保证 set 的迭代顺序；特别是它不保证该顺序恒久不变。此类允许使用 null 元素。线程不同步

剖析源码：

private static final Object PRESENT = new Object();


public HashSet() {
    map = new HashMap<>();
}


public boolean add(E e) {
    return map.put(e, PRESENT)==null;
}

public boolean remove(Object o) {
     return map.remove(o)==PRESENT;
}
总结： 1.hashSet底层实现是 hashmap 对set的一切操作实质都是在操作map 2.之所以set存放的元素是不重复的，从源码可以知道 当向set中add一个元素时，是在向map中put一个键值对，key 就是参数 e value 是一个固定的object对象 所有的value都是一个值。而map中 key又是唯一的，当有相同的key 被put的时候，会覆盖掉之前的key 。所以当add一个已经存在的元素时，map.put()返回的是旧的value值。 3.remove同理。因为value值都是固定的Object 且是final修饰的，不会增加额外的内存开销。
  • LinkedHashSet

    是HashSet的子类，但是维护了一个双向链表，所以add remove等操作更高效

  • TreeSet

    基于 TreeMap 的 NavigableSet 实现。使用元素的自然顺序对元素进行排序，或者根据创建 set 时提供的 Comparator 进行排序，具体取决于使用的构造方法。

集合框架底层数据结构总结

Collection

1.List

  • ArrayList : Object数组
  • Vector : Object数组
  • LinkedList : 1.6之前是双向循环链表 1.7之后取消了循环 变成了双向链表

2.Set

  • *HashSet（无序，唯一）: * 基于hashMap来实现的，底层用hashMap来保存元素
  • *LinkedHashSet : * LinkedHashSet 继承于HashSet，并且其内部通过LinkedHashMap来实现。
    有点类似于LinkedHashMap其内部是基于HashMap实现一样，不过还是有点儿区别。
  • TreeSet（有序，唯一）： 红黑树（自平衡的排序二叉树）

3.Map

  • HashMap： JDK1.8之前HashMap由数组+链表组成的，数组是HashMap的主体，链表则是主要为了解决哈希冲突
    而存在的（“拉链发”解决冲突）。JDK1.8以后再解决哈希冲突时有了较大的变化，当链表长度大于阈值（默认为8）
    时，将链表转化为红黑树，以减少搜索时间。
  • LinkedHashMap： LinkedHashMap继承来自HashMap，所以它的底层仍然是基于拉链式散列结构即由数组和链表
    或红黑树组成。另外，LinkedHashMap在上面结构的基础上，增加了一条双向链表，使得上面的结构可以保持键值对的插入
    顺序。同时通过对链表进行相应的操作，实现了访问顺序相关逻辑。详细可以查看
    《LinkedHashMap 源码详细分析（JDK1.8）》
  • Hashtable： 数组+链表组成的，数组是Hashtable的主体，链表则是主要为了解决哈希冲突而存在的
  • TreeMap： 红黑树（自平衡的排序二叉树）

如何选用集合？

主要根据集合的特点来选用，比如我们需要根据键值获取到元素值时就选用Map接口下的集合，需要排序时选择
TreeMap，不需要排序时就选择HashMap，需要保证线程安全就选用ConcurrentHashMap.当我们只需要存放元素时，就
选择实现Collection接口的集合，需要保证元素唯一时选择实现Set接口的集合比如TreeSet或HashSet，不需要就选择实
现List接口的比如ArrayList或LinkedList，然后再根据实现这些接口的集合特点来选用
