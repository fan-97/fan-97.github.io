平凡的秋招

2019-09-26

从8月4日到今天（9月26日）我的秋招总算是结束了，想总结一下整个过程一些学习和面试的知识。（主要是吹牛）

在大三快结束的时候，学校来了一些培训机构（其实就一两个） 吹嘘着什么培训完包就业、在成都第一年年薪6w+（对于一个即将毕业（失业）的人，这个就仿佛是面前有100块钱，忍不住不捡）当时就拉着小伙伴一起去报名了，然后跟他又一阵交流（继续吹嘘）更加坚定了我要去培训的信念。然后去了培训的地方，第一天开会没开空调把我们热晕了，一回想起在家里面可以每天吹空调，我们就下定决心不去上课，让培训机构退报名费。因为开训之前讲了一些关于课程的事情，所以我就根据他的时间制定了我自己的学习时间表。

因为培训费用要2W，我想着我这么穷怎么可能有这么多钱，就下定决定要自学，但是又没有目标，很盲目，就网上四处搜寻资料，百度(java后端如何准备秋招？Java程序员应该会什么？)。所以在这里要推荐一个很好的网站牛客网，当时就是在里面找到一些学习的方向。

校招跟实习的区别就在于 校招的offer是正式offer 所以薪水也有不小的差距 拿到offer过后一般也会有一个实习期的，但是会签订三方协议，如果是实习offer一般是不会签三方协议的。

参加校招必须具备的技能（Java开发）

  • Java基础：
    集合(ArrayList、LinkedList、HashMap、TreeMap、LinkedHashMap、ConcurrentHashMap等，尽量多去看源码学习)
    多线程（线程的创建方式、线程可能会造成的问题[死锁]、线程之间通信、synchronized、volatile、轻量锁、偏向锁等各种锁）
    并发 反射、IO JVM（垃圾回收、类加载机制、Java内存模型、调优等 推荐书：《深入理解Java虚拟机》)

  • mysql：
    sql语句 索引 调优

  • noSql:
    Redis

  • 框架：
    Spring SpringMVC Mybatis Spring Boot Spring Cloud(看源码、多写项目)

  • 数据结构：
    数据结构对校招来说是非常重要的，因为你没有太多的项目经验 如果你具备基本算法思想的话，那么对以后的开发工作 就能做到更高效。推荐一本书 《大话数据结构》

  • 设计模式:
    设计模式在开发中，可以让你的代码更高效，耦合度更低 复用性更高。主要要学习 工厂模式 代理模式 单例模式 这3种设计模式 在Spring里面都有使用过，所以一定要认真学习，理解。

  • 计算机网络：
    计算机网络在面试中也是必考、必问的，TCP/IP协议、TCP、UDP区别等等。而且互联网公司 你不了解这些网络协议的话 在开发中也是会遇到很多困难的 推荐一本书 TCP IP详解 卷一 看完卷一就差不多能够对付这个校招了，要想深入学习就继续学 卷二 卷三。

  • Linux：
    不要觉得开发Java就不会接触到Linux了，现在公司服务器基本上都是部署在linux上，所以会linux 的一些基本的命令 能够部署项目到服务器上并发布就差不多了。毕竟很多公司都是有专业的DBA和linux运维工程师的

学这些知识的时候一定要多动手，不要只会敲一些demo就行了，毕竟以后工作不是让你做demo而是让你解决一些实际的问题，所以一定要运用到实际的项目中。学完这些还是需要一定的时间的，所以秋招一定要迟早准备 越早准备越好 毕竟可以多复习几遍，有些东西不是学一遍就能够给记住的 要想去大公司，还得会《操作系统》、《计算机组成原理》
